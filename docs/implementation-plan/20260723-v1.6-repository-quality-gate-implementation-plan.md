# v1.6 Repository Quality Gate Implementation Plan

Date: 2026-07-23

Status: Implemented and validated; merge pending

## Objective

Make calibration's first-party executable and documentation contracts fast,
deterministic, and reproducible through one local and CI command. This is a
repository-local response to observed validation gaps, not a mandatory
cross-project tool checklist.

## Scope

- Make `pyproject.toml` and committed `pdm.lock` the sole dependency authority
  for development and validation.
- Use pytest for all Python and installer behavior tests.
- Require 100% line coverage for first-party Python under `scripts/`.
- Use Ruff for Python lint and formatting checks and Pyright strict mode for
  first-party Python and tests.
- Use rumdl for first-party Markdown lint and a repository-owned validator for
  local Markdown link targets.
- Use ShellCheck and `bash -n` for the installer, plus isolated behavior tests
  for dry-run, idempotence, conflict, backup, retirement, force, and paths with
  spaces.
- Keep the GitHub Actions job and stable workflow check name
  `validate-skills`.

## Exclusions

- Do not change skill instructions, triggers, or behavioral evaluation cases.
- Do not lint or rewrite vendored `thirdparty/` content.
- Do not add a publishable Python package, Node project, pre-commit framework,
  or a second test framework.
- Do not claim model or workflow behavior improvement from static validation.

## Acceptance

- `pdm lock --check` succeeds.
- `pdm run check` succeeds on Python 3.12 and the supported local Python.
- All first-party Python lines are covered by pytest.
- Installer dry-run against an explicit temporary `CODEX_HOME` succeeds.
- Worktree, staged, and committed branch-range diff checks succeed.
- GitHub Actions publishes a successful `validate-skills` check before any
  ruleset change.
