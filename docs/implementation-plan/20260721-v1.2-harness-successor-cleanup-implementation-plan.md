## Slice 2: Staged Comparative Evaluation

Status: pending Slice 1 commit

Freeze the accepted baseline and successor source with the same model, medium
reasoning effort, fixtures, turn protocol, three repetitions, isolated
per-session homes, system-skill snapshot, seeded schedule, and blind mapping.
Precompute and fingerprint the complete 19-case run before executing either
stage.

### Stage A: targeted cases

Run the six harness and evaluation cases for both arms: 36 sessions and 18
blind pairs.

Continue only when:

- the successor has no critical failure in the 18 outputs;
- all three `harness-human-escalation` outputs pass their critical gate;
- successor required-behavior coverage is not below the baseline; and
- successor pairwise wins are not fewer than baseline wins.

A failed Stage A closes the candidate without running regression cases.

### Stage B: regression cases

After Stage A passes, run the remaining 13 cases for both arms: 78 additional
sessions and 39 blind pairs. Together both stages form the complete 114-session
run.

Accept regression only when:

- no output introduces a successor critical failure where its baseline pair
  passes;
- successor critical failures do not exceed baseline critical failures;
- successor required-behavior coverage is not below the baseline; and
- successor pairwise wins are not fewer than baseline wins.

Time, turns, and token usage remain secondary measures and cannot override a
quality-gate failure.

### Blind review

A fresh independent task receives only blind pairs, rubrics, the blind
manifest, and an empty scorecard. Validate row count, uniqueness, required
fields, score bounds, and notes before copying the scorecard to the canonical
results tree or revealing the arm map.

## Delivery Decision

- Keep the candidate pull request draft while either evaluation stage is
  incomplete.
- If a gate fails, do not merge the candidate. Close it with a separate
  docs-only decision recording the stopped stage, source commits,
  fingerprints, results, and limitations.
- If both stages pass, the evaluation authorizes merging the candidate. Then
  publish a separate docs-only closeout and mark this plan closed.
- Keep Slice 4 closed until that closeout explicitly retains the successor and
  confirms no critical regression.
